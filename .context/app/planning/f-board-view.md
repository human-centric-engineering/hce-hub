## Intent

The **Board** — what's in flight *right now, by person*: a 2-D matrix of **member swim lanes** (rows) × **effective-status columns** (Available · Claimed · In PR · Merged · Backlog). It renders the *same* seeded plan as [[f-plan-view]], task-centric instead of feature-centric. The design centre is **pull-not-push** (§5): every task shows in its owner's lane but anyone can claim; deps-blocked work recedes to Backlog; overlap is a **soft, ambient collision marker**, never a lock (§5/§13.5). Fills the container's Board tab (the §09 placeholder).

## Reconciliation with current repo reality   (required — done first)

Verified against `main` after f-plan-view shipped, 2026-07-15 (recon over `lib/projects/{plan,plan-order,task-status,collision,access,user-refs}.ts`, `lib/projects/capabilities/claim-task.ts`, `components/hub/projects/{project-view,plan/**}`, `app/(hub)/projects/[id]/page.tsx`, `prisma/schema/app.prisma` `TaskClaim`/`Task`, `design/board.jsx` + README §3). Each finding is a decision.

- **No board read API exists → build `GET /api/v1/projects/[id]/board` (mirror of `/plan`).** The board needs data `/plan` deliberately omits: **member lanes** (rows), **soft-collision** flags (from open `TaskClaim` file overlap — not in `/plan`), and **is-mine** (the caller). A dedicated endpoint shaped for the view (exactly as §09 built `/plan`) is cleaner than overloading `/plan` with unused board fields. → **Decision:** `lib/projects/board.ts` `getProjectBoard(userId, projectId)` (`withAuth`, `parseCuidParam`); routes through `getAccessibleProject` → **404-not-403** (reuse the funnel). Returns the board **fully shaped server-side** (lanes with routed/columned tasks + totals) so the client stays dumb and the load-bearing routing is unit-testable at the boundary — the same server-does-the-logic call as `planOrder()` in §09.
- **Routing = `computeEffectiveStatus` (reuse) + `claimer ?? owner` lane.** Each task lands in exactly one cell: **lane** = its effective claimer (`claimedByUserId` when effectively claimed) else its `feature.ownerUserId` ("unclaimed routes to the owner's lane — no separate unclaimed lane", README §3); **column** = its effective status. → **Decision:** reuse `lib/projects/task-status.ts` `computeEffectiveStatus` (so Plan/Board never diverge); map effective `blocked` → the **Backlog** column (the design's "available-with-unmerged-deps → Backlog"). **Column totals count *effective* status** (fixing the design mock's raw-status header counts, which disagree with the effectively-routed cells).
- **[carried-in — `f-data-model` t-2 `/code-review`] Null-claimant `claimed` → route to the owner lane, not a phantom lane.** A task with `status='claimed'` but `claimedByUserId=null` (erased claimant) is effectively **available** (`computeEffectiveStatus`) → it routes by `owner`, lands in its owner's lane's Available column, and is re-pullable. The routing key is `effectiveClaimer ?? ownerUserId`, never a raw `claimedByUserId` that could be null. (Found building [[f-data-model]] t-2.)
- **[carried-in — `f-data-model` t-3 `/code-review`] An "Unassigned" lane absorbs orphaned tasks; never deref a null/absent ref.** A task whose routing target (claimer or owner) is **null** (erased/never-set) **or not a current project member** (owner left the project, row retained) has no member lane. → **Decision:** a terminal **"Unassigned" lane** (the design's dashed `UnassignedLane`, repurposed) catches these so nothing is dropped or NPE'd; lane heads, claimer avatars, and cards render a null ref as "unassigned/former member" (reuse `plan/**`'s null-safe `initials`/avatar pattern). Members (the real lanes) come from `ProjectMember`; the routing target must be in that set or it falls to Unassigned. This discharges the null-render finding at the board level.
- **Soft collision = reuse `filesOverlap` over open `TaskClaim`s (dormant on seed, real when claims exist).** `lib/projects/collision.ts` (`filesOverlap`/`pathsOverlap`) + the `claim_task` query pattern give the overlap logic. → **Decision:** the endpoint computes per-task collision flags server-side — gather open claims (`TaskClaim releasedAt=null`) in the project with their task's `filesScope`, mark any pair with `filesOverlap` as colliding (note: "overlaps '<other task>' claimed by <user>"). The seed sets no `filesScope`/`TaskClaim` rows, so markers are **absent on seed data** (like PR links) — a real capability, dormant until claims overlap. Ambient rendering only (§13.5): a quiet marker + slow pulse, never a lock.
- **The container + `?view=board` scaffold exist — fill the board branch; page fetches `/board` in parallel.** `project-view.tsx` renders `BOARD_COPY` on the board tab; `[id]/page.tsx` fetches the header (+ plan on the plan tab). → **Decision:** page fetches `/board` when `view==='board'` (parallel with the header, mirroring §09's plan fetch — no waterfall); `project-view.tsx` mounts `<BoardView board={board} />` on the board tab, graceful message on fetch failure. Reuse the `plan/**` client patterns (StatusPill, `presentation` tones, DTO-mirror `types.ts`, `initials`, **`sanitizeUrl` on the card's PR link**). New components under `components/hub/projects/board/**`. **No new guard** (group auth + `/projects` edge gate cover it). **No HB2** (fork-owned files).
- **Shared feature/task load — factor if clean, else load independently.** `plan.ts` and `board.ts` both read features+tasks+refs+`computeEffectiveStatus`. `computeEffectiveStatus` is already the shared primitive (the thing that must not diverge). The Prisma `select` overlaps but the shapes differ (plan = features→tasks tree; board = flat tasks + lanes + collision). → **Decision (build-time):** prefer a small shared internal task-loader in `lib/projects/` if it lands cleanly without contorting the shipped `plan.ts`; otherwise `board.ts` loads independently (minor `select` duplication is acceptable — the divergence risk lives in `computeEffectiveStatus`, already shared). Don't refactor shipped §09 code just for DRY.

**Tier / seam hypotheses (confirmed):** pure leaf-app, **zero platform edits, zero upstream asks**. New fork-owned `GET /api/v1/projects/[id]/board` (GET, rate-limit automatic), `lib/projects/board.ts`, `components/hub/projects/board/**`. Live reused deps: `access.ts` (404-not-403), `task-status.ts` (`computeEffectiveStatus`), `collision.ts` (`filesOverlap`), `user-refs.ts` (`fetchUsers`), `sanitize.ts` (`sanitizeUrl`), the `plan/**` presentation pattern — none re-implemented. **No migration** (all models shipped in f-data-model → `/pre-pr` drift step N/A). **No HB2 / no `platform-divergences` row.**

## Test strategy

vitest = happy-dom, no live DB ([[planning-retro]] B9): mock `@/lib/db/client` / the funnel; components via `@testing-library/react`; `serverFetch`-mocked page smokes.

- **The load-bearing tests are the routing + the membership 404-not-403 (t-1).** `board.test.ts`: a `claimed` task routes to the claimer's lane + Claimed column; an `available` task routes to the **owner's** lane; a deps-blocked `available` → **Backlog** (via `computeEffectiveStatus`); a **null-claimant `claimed` → owner lane, Available** (carried f-data-model t-2); an **orphan (null/non-member owner) → Unassigned lane** (carried f-data-model t-3, never throws); column totals are effective-status counts; two open claims with overlapping `filesScope` flag both tasks (`filesOverlap`); a **non-member/unknown id → 404, never 403** (re-assert anti-enumeration at the boundary).
- **UI (t-2):** lanes render in task-count order + the Unassigned lane; a card shows title/ref/claimer/PR (sanitized); `is-mine` border; the collision marker renders when flagged; a null claimer/owner renders the fallback, never `undefined.name`; empty columns show the dot; `serverFetch`-mocked page smoke (default header + `?view=board` fetches `/board` and renders a lane).
