# f-selfhost-cutover — import `plan.md` → Hub; the Hub becomes its own system of record

*Feature 19 on [[plan]] — the **third and final** feature of the self-hosting cluster (§17 [[f-journal]] → §18 [[f-feature-planning]] → §19). Binding design: [[self-hosting#5 · How features get into the Hub — three paths|self-hosting §5]] (path C, the coded import) + [[self-hosting#6 · Feature breakdown & sequencing|§6]]. Depends on [[f-feature-planning]] (features materialise through its `Feature`/`IndicativeTask`/`Task` model — shipped) and [[f-projects]] (the project the import populates — shipped). Built via the **current GitHub flow** — the last thing built the old way; **from the moment this ships, §12–§15 are managed in the Hub over MCP.** Also **folds in [[plan#Parked phases (future epics)|`f-project-slugs`]]** (owner, 2026-07-21) — shareable, durable project URLs matter once the Hub *is* the record.*

## Intent

The dogfood switch — the Hub becomes **the record of building the Hub**, testing its own flows and capabilities on itself. **Scope is explicitly the Hub-on-itself:** onboarding other studio projects (Lelanea, …) is *not* part of this cutover — no other project is in play at this stage. Three moves:

1. **A durable, portable transfer mechanism for a project's coordination graph** — `app:project:export`/`import`, a versioned JSON snapshot (mirroring Sunrise's `lib/orchestration/backup/*`) that preserves ids + `createdAt`. This is the answer to *"how do we get the data in and out"* (owner decision, 2026-07-21): the Hub is now the **system of record**, so its live state must survive `db:reset` (export → import) and move **dev → prod** (export → import) — a seed can't do either.
2. **The backdated cutover load** — a one-shot command that materialises *this* `plan.md`'s full history into the Hub (all 19 features with descriptions/done-whens/indicative sketches/deps; shipped features' tasks → `merged` `Task` rows with PR URLs; the **decisions log** → backdated `decision` events; **work-completed** → `feature_shipped` narratives), then **retires `006-sample-plan`** (a per-reset seed would clobber the live record).
3. **Project slugs** — `/projects/hce-hub` instead of `/projects/<cuid>` (the feature-page slug pattern, one level up), so a shared project link is durable and human.

After it ships: `plan.md`/`<feature>.md` **freeze** into history; the MCP dev loop + the export/import backup ritual are documented; every claim/plan/ship/decision is an MCP call (or a UI action — §18 model correction).

## Test strategy

vitest = happy-dom, no live DB ([[planning-retro]] B9): mock `@/lib/db/client` / `tx`; the migration's `Project.slug @unique` is proven by `db:drift-check` (not vitest); scripts are thin CLIs — test the `lib/projects/transfer/**` **pure** serialize/deserialize + `importProject` upsert logic against a mocked prisma, and prove real-DB fidelity with a `smoke:*` script (mirroring `app:smoke:erasure`).

- **Transfer (t-1, load-bearing):** **round-trip identity** — a fixture project graph → `export` → `import` (into a fresh mocked store) → `export` produces byte-stable JSON (ids + `createdAt` preserved); a member whose `userId` doesn't resolve → an `ImportResult` warning, no `User` created; a `schemaVersion` mismatch is rejected; an empty/partial graph imports safely (safe-on-empty). A real-DB smoke: export the 006-seeded project, wipe, import, assert equality.
- **Cutover load (t-2):** the pure snapshot builder — every shipped feature has a `feature_shipped` event with the right backdated `createdAt`; a Decisions-log entry naming a feature slug scopes its `decision` event to that `featureId`, an architectural one is project-scoped; shipped tasks carry PR URLs + `merged`; `import-plan` is idempotent (a second run updates, doesn't duplicate — upsert by stable id); the builder is `buildSamplePlan`'s moved test, extended.
- **Slugs (t-3):** the detail route resolves a slug within scope and a cuid; an unknown slug/cross-member → 404; the backfill derives a stable slug from the name; `updateProject` changing `name` leaves `slug` unchanged; authoring rejects a duplicate slug (`@unique`).
