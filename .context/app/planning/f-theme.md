# f-theme ‚Äî HCE Hub base theme

*Feature 02 on [[plan]]. Binding design: [[design_handoff_hce_hub/README|design handoff]] "Design system / tokens" (the full token set + `styles.css` source-of-truth incl. `.theme-dim`); [[v1-requirements#13.5 Tone, feel, and anti-patterns|¬ß13.5]] (warm/low-chroma, border-not-shadow, no celebratory motion, "sibling to Sunrise admin"). Depends on [[f-fork]] (the auth-only shell + `BRAND`). First UI-spine feature under the owner's usable-first build order ([[plan#Decisions log|plan ¬ßDecisions 2026-07-14]]).*

## Intent

The **token foundation** every later UI-spine feature (`f-shell`, `f-projects`, the boards, the sidekick) paints on: the warm, low-chroma HCE palette + type + motion system from the [[design_handoff_hce_hub/README|design handoff]], landed **once** as a fork-owned CSS layer so shadcn primitives restyle by inheritance and Hub-native surfaces have a vocabulary of tokens (`--accent` clay, `--signal-*`, mono/sans) to consume. It ships the theme *foundation* + the auth-page paint + the "H" brand-mark; **per-screen styling lands with each consuming feature** (design README; [[plan]] ¬ß138). No new route, no new data ‚Äî its job is to make "warm/dim HCE" the default look of every Hub (non-`/admin`) surface without editing a single platform file.

## Reconciliation with current repo reality   (required ‚Äî done first)

Verified against `main`, 2026-07-14 (recon over `app/globals.css`, `app/brand-theme.css`, `lib/app/surface.ts`, `app/layout.tsx`, `components/brand/brand-mark.tsx`, `lib/security/headers.ts`, `.context/ui/surface-theming.md`, the design handoff `styles.css`). Each finding is a decision.

- **Watch-item A ("theme sync-safety") resolves BETTER than the plan hypothesized ‚Äî ZERO platform-file edits, no keep-mine, no ledger row.** [[plan]] ¬ß80/¬ß85 assumed a full re-theme touches `app/globals.css` (platform) as a "keep-mine" merge. It does **not**: Sunrise ships a purpose-built, fork-owned theming seam ‚Äî `app/brand-theme.css` (ships empty, imported in `app/layout.tsx` **after** `globals.css`) + `lib/app/surface.ts` (`classifySurface` policy) + the `data-surface` proxy/`<SurfaceSync>` plumbing (upstream [#355](https://github.com/human-centric-engineering/sunrise/pull/355); see `.context/ui/surface-theming.md`). ‚Üí **Decision:** the **entire** HCE theme lands in `app/brand-theme.css`. `globals.css` stays byte-for-byte untouched, so upstream palette tweaks still flow through every token we don't override. Watch-item A is **closed with no upstream ask and no platform touch** ‚Äî strictly cheaper than the plan's "keep-mine" hypothesis.
- **Surface scope = `consumer` only (owner-confirmed 2026-07-14).** `classifySurface` already returns `consumer` for every non-`/admin` path and `admin` for `/admin[/‚Ä¶]`. The Hub is auth-only, so "consumer" = the whole Hub working surface; `/admin` (where `f-project-admin` ¬ß13.2 lives) stays on Sunrise defaults ‚Üí the Hub reads as a **sibling to Sunrise admin** (¬ß13.5, "same family, different room"). ‚Üí **Decision:** scope every override to `[data-surface='consumer']` (+ the `.dark` **compound** form for dim). **`lib/app/surface.ts` is NOT edited** ‚Äî the default policy is already exactly what we want, so no HB2 default-test break there either. (Collapsing admin into the theme would be a one-line `classifySurface` change later if ever wanted ‚Äî deliberately not done.)
- **Two-layer token model (the shadcn tokens are necessary but not sufficient).** `globals.css`'s `@theme` defines the shadcn semantic tokens (`--color-background/-foreground/-primary/-muted/-border/-card/‚Ä¶`) Tailwind turns into utilities (`bg-background`, `text-foreground`). The design handoff uses its **own** richer vocabulary (`--bg`, `--ink`, `--ink-mute/-faint/-ghost`, clay `--accent`, six `--signal-*`, `--line-*`, radii/shadow/motion, `--font-ui/-mono`). ‚Üí **Decision:** `brand-theme.css` carries **both**, unlayered, under the consumer scope: **(a)** remap the shadcn tokens onto the warm palette (`--color-background`‚Üí`#faf8f3`, `--color-foreground`‚Üí`#1a1a1a`, `--color-primary`‚Üíink for the ink-primary-button look, `--color-border`‚Üí`--line`, `--color-muted`/`--color-card`/`--color-popover`‚Üíthe warm tint set) so **every shadcn primitive restyles for free by CSS-var inheritance** (the design's "restyle shadcn to the token set" requirement ‚Äî no component edits); **(b)** add the Hub-native tokens shadcn has no slot for (`--accent`/`--accent-bg`/`--accent-ink` clay, `--signal-*` + `-bg`, `--ink-mute/-faint/-ghost`, `--line/-soft/-strong`, `--radius-*`, `--shadow-*`, `--duration-*`/`--ease`, `--font-ui`/`--font-mono`), consumed **directly** by Hub components later. **Do NOT** remap shadcn's `--color-accent` to clay ‚Äî shadcn uses `accent` for muted hover fills; clay is a *separate* sparingly-used token, else every hover turns clay.
- **Fonts: self-host `.woff2` + `@font-face` in `brand-theme.css` ‚Äî CSP forces this, and it's the zero-platform-edit path.** No `next/font` exists anywhere in the tree (Sunrise runs on system fonts), and `lib/security/headers.ts` sets **`font-src 'self' data:`** and **`style-src 'self' 'unsafe-inline'`** ‚Äî so a Google-Fonts `@import` (needs `fonts.googleapis.com` in `style-src` + `fonts.gstatic.com` in `font-src`) is **CSP-blocked**. ‚Üí **Decision:** vendor Inter Tight (400/450/500/600) + JetBrains Mono `.woff2` into `public/fonts/`, declare `@font-face` in `brand-theme.css`, point `--font-ui`/`--font-mono` at them. Served from `'self'` ‚Üí CSP-clean, **no `headers.ts` edit, no platform touch.** *Alternative considered & rejected:* `next/font/google` (idiomatic, self-hosts at build) needs its generated `.variable` class on `<html>`/`<body>` ‚Üí a platform `layout.tsx` edit; self-hosting avoids it for the cost of committing the woff2 files. (Weights: 4√ó Inter Tight + 2√ó JetBrains Mono (400/500) ‚âà 6 files; `font-display: swap`.)
- **Brand-mark "H" lands here (deferred from `f-fork`).** `components/brand/brand-mark.tsx` is the fork-owned scaffold (returns the bare `BRAND.name` string today; Sunrise never changes its body ‚Üí clean-merging). The design mark is a 26px `--ink` square, `--bg` "H", radius 6px, mono 600. ‚Üí **Decision:** render `bg-foreground text-background` (which resolve warm on the consumer surface) at the design's size/radius; keep `BRAND.name` as `aria-label` (the stable accessibility/identity string). This is a **render-concern** edit to a file *designed* to be forked ‚Äî not a platform divergence.
- **What f-theme does NOT chase.** The protected *chrome* (three-column shell) is `f-shell`; f-theme themes the **auth pages** (login/signup/reset ‚Äî exist from f-fork, on the consumer surface) which inherit the remapped shadcn tokens automatically. If a platform auth page hardcodes a colour instead of using a semantic token, **leave it** ‚Äî do not fork-edit a platform page for pixel-parity; it gets styled when its owning feature touches it. f-theme's proof-of-life is: auth pages read warm, shadcn primitives inherit the tokens, the "H" renders, dim mode flips correctly.

**Tier / seam hypotheses (confirmed):** pure leaf-app. Fills the fork-owned `app/brand-theme.css` seam (its designed purpose) + edits the fork-owned `brand-mark.tsx` scaffold + adds `public/fonts/` assets. **Zero core‚Üífork seams, zero upstream asks, zero platform-file edits** (watch-item A closed). No `platform-divergences.md` row ‚Äî nothing platform-owned is touched.

## Test strategy

Theme work is **CSS-shaped**, not logic-shaped ‚Äî most of it is not vitest-testable, and manufacturing assertions over CSS variables would be brittle theatre ([[planning-retro]] ‚Äî don't test the framework). The real proof is **visual + CSP-console + a zero-platform-edit diff check**, run by the owner at review.
- **Automated (the one real unit):** `brand-mark.test.tsx` ‚Äî renders `<BrandMark>`, asserts the "H" glyph is present and `BRAND.name` is the accessible name (`aria-label`). This satisfies `/pre-pr` 4f (the only changed `.tsx`).
- **Manual at review (documented in the PR):** (1) auth `/login` renders warm; (2) toggle `.dark` ‚Üí dim (warm-dark `#1a1916`, not blue-black); (3) `/admin` still renders Sunrise-default (surface split holds); (4) DevTools console shows **no** CSP `font-src`/`style-src` violation and both families load from `/fonts/‚Ä¶` (not gstatic); (5) `git diff --stat` lists **only** `app/brand-theme.css`, `public/fonts/**`, `components/brand/**`, `tests/**`, `.context/app/**` ‚Äî no platform path.
- **No smoke** ‚Äî nothing DB- or request-referential to prove end-to-end (contrast f-data-model's `app:smoke:erasure`).
